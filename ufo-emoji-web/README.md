# 🛸 UFO Emoji — WebAssembly Edition 👽

A browser port of Todd Bruss' iOS SpriteKit arcade game **UFO Emoji**, compiled
to WebAssembly with **[SuperBox64Kit](https://github.com/SuperBox64/SuperBox64Kit)**
(a Swift reimplementation of Apple's SpriteKit that targets WASI Preview 1) and
driven in the browser by **[WasmKit](https://github.com/SuperBox64/WasmKit)**
(Canvas2D renderer + Web Audio mixer + DOM/gamepad input). No Emscripten, no
loading screens, no watermarks.

Every actor in the game is drawn as **live emoji text** — the runtime renders
`SKLabelNode` glyphs natively on Canvas2D, so the ship, enemies, power-ups,
scenery and explosions are all real emoji, exactly in the spirit of the
original.

```
                🛸 UFO EMOJI 👽
        an emoji arcade shooter on WASM
```

## Play

Serve the `web/` folder over HTTP and open `index.html`:

```bash
cd web
python3 -m http.server 8000
# open http://localhost:8000/
```

(`file://` won't work — the runtime fetches the wasm + assets.)

### Controls

| Action | Keyboard | Touch / Mouse | Gamepad |
|---|---|---|---|
| Fly | Arrows / WASD | Drag (virtual flight yoke) | Left stick |
| Fire | Space (hold) | Hold | A / Right trigger |
| Start / Restart | Space | Tap | A |
| Switch pilot (title) | 1 · 2 · 3 | Tap the pilot line | — |

## Gameplay

- **Three pilots:** 👽 Alien (🛸), 🐵 Monkey (🚀), 💩 Poo (🚀) — pick on the title screen.
- **Three worlds** scroll past as you climb the 12 levels:
  - 🌊 **Water World** (levels 1–4)
  - 🏜️ **Sand Dunes** (levels 5–8)
  - 🌌 **Outer Space** (levels 9–12)
- **Blast the enemy emoji**, dodge their fire, and clear a kill quota to advance.
- **Boss fights** (👑) every 4th level — heavily armored, lots of points.
- **Friendly drifters** (🐬 🦋 🦄 …) float by for bonus points if you tag them.
- **Power-ups** drop from wrecks:
  | Glyph | Effect |
  |---|---|
  | 🔫 | Double laser |
  | 💠 | Rapid fire |
  | 🛡 | Temporary shield |
  | 🔱 | Smart bomb (clears the screen) |
  | ❣️ | Extra life |
  | 💎 | Bonus points |
- **Extra life** every 5,000 points. High score persists via `localStorage`.

## Build

Requires the swift.org **Swift 6.3.2** toolchain + the **`swift-6.3.2-RELEASE_wasm`**
SDK, and (optionally) `wasm-opt` from binaryen for the smallest binary.
[WasmKit](https://github.com/SuperBox64/WasmKit) is expected as a sibling
checkout (`../../WasmKit`); the build script clones it if missing.

```bash
./build.sh release    # -> web/ufoemoji.wasm  (+ runtime.js + manifest.json)
./build.sh debug      # fast, unoptimized
```

`build.sh` compiles the reactor, publishes it into `web/`, regenerates
`manifest.json` from `web/assets/`, and copies WasmKit's `runtime.js`.

## How it maps to the original

The original ([`../UFO Emoji`](../UFO%20Emoji)) is a UIKit app whose levels are
binary `.sks` scene files driven by `SKTileMapNode` tilemaps — neither of which
the wasm SpriteKit subset loads. So the side-scrolling gameplay is re-expressed
as a single code-driven `GameScene` that keeps `import SpriteKit` unchanged
(same approach as [AsteroidZ](https://github.com/SuperBox64/AsteroidZ)'s web
port). The pilots, enemy/goodie rosters, three worlds, power-ups, scoring tiers,
boss "King/Queen" fight, flight-yoke feel and Ready·Set·Go intro all come
straight from the original's `GameGlobal`, `GTFlightYoke`, `GameBadGuyAI`,
`GameWorld` and `GameScene` sources. Sound effects and music are the original
GarageBand assets, transcoded to `.wav`/`.ogg` for the Web Audio pipeline.

## Layout

```
ufo-emoji-web/
├── Package.swift              # depends on SuperBox64Kit (branch: embedded)
├── build.sh                   # swift build + wasm-opt + publish to web/
├── Sources/UFOEmoji/
│   ├── main.swift             # reactor entry: _initialize / boot / frame
│   └── GameScene.swift        # the whole game
├── tools/
│   ├── smoketest.mjs          # headless: instantiate wasm, drive input, assert no trap
│   └── shoot.mjs              # Playwright: screenshot title + gameplay
└── web/
    ├── index.html             # host page (window.WASMWEB config)
    ├── ufoemoji.wasm          # built artifact
    ├── runtime.js             # WasmKit runtime (copied by build.sh)
    ├── manifest.json          # asset list (generated by build.sh)
    └── assets/sfx/            # *.wav SFX + music.ogg
```

## Tests

```bash
node tools/smoketest.mjs       # boots the wasm headless, feeds synthetic key
                               # events, runs ~1000 frames, asserts it renders
                               # text/sprites + plays sound and never traps
```
